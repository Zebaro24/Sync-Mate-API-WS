# ADR-0003: WS-протокол — общий контракт между бэкендом и расширением

- **Статус:** Accepted
- **Дата:** 2026-06-27 (зафиксировано постфактум)

## Контекст

Синхронизация воспроизведения работает через набор WebSocket-сообщений (`connect`, `info`, `play`,
`pause`, `seek`, `status`, `load`, `set_video`, `remove_block_pause`). Бэкенд и расширение —
два независимых репозитория. Если набор/поля типа сообщения разойдутся, синхронизация ломается
**молча** (исторический баг: action в `_VALID_ACTIONS` без обработчика «принимался» без эффекта).

## Решение

Считать набор WS-типов единым контрактом. Источник на бэке: `UserHandler._VALID_ACTIONS` +
все `send_json({"type": ...})` + `app/ws/schemas.py`. На фронте: `WSMessageTypes`. Оба должны
совпадать побитово по множеству типов. См. [../websocket-protocol.md](../websocket-protocol.md).

## Последствия

- ➕ Чёткая, проверяемая граница между двумя репозиториями.
- ➖ Любое добавление/переименование типа = правка **обеих** сторон одновременно + обоих
  `websocket-protocol.md` + `DOCUMENTATION.md`.
- 🔒 Enforced: `python scripts/protocol_sync.py` (гейт-проверка `protocol`, гоняется в **обоих**
  гейтах) падает при расхождении; хук `guard-protocol` напоминает при правке файла протокола;
  скил `/sync-protocol` и агент `protocol-guardian` диктуют правки обеих сторон.

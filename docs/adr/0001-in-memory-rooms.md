# ADR-0001: Комнаты хранятся в памяти, без персистентности

- **Статус:** Accepted
- **Дата:** 2026-06-27 (зафиксировано постфактум)

## Контекст

Комнатам нужно состояние (участники, текущая позиция, флаги паузы/загрузки). Вариантов хранения два:
внешнее (Redis/БД) или in-process. Проект — небольшой сервис синхронного просмотра с короткоживущими
комнатами; простота важнее durability.

## Решение

Хранить комнаты в модуль-глобальном in-memory словаре (`RoomService._storage`, синглтон через
`app/modules/room/dependencies.py`). Никакой персистентности. См. [../architecture.md](../architecture.md) §10.

## Последствия

- ➕ Минимум зависимостей, простой код, мгновенный доступ.
- ➖ Перезапуск API **стирает все комнаты** — это ожидаемо, не баг.
- ➖ **Один процесс**: состояние не разделяется между воркерами uvicorn. Поэтому запуск без `--workers`
  (Docker/compose поднимают один процесс). Не добавляй `--workers > 1` без внешнего хранилища.
- ➖ Нет автоудаления пустых комнат — только ручной `DELETE` через REST.
- 🔁 Если понадобится горизонтальное масштабирование/персистентность — заменить `_storage: dict` на
  Redis-обёртку (новый ADR, Supersedes этот).

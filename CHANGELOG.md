# Changelog

## v0.2.3 — 2026-06-27

- Выход «тормозящего» участника пересчитывает готовность (`remove_user`) — остальные не замерзают навсегда
- Таймаут гейта готовности (`READY_TIMEOUT`) — один медленный клиент не блокирует комнату; в статус уходит «кого ждём»
- `set_video` обнуляет буферы участников — нет ложной готовности при смене ролика
- Рассылка seek отстающим вынесена из-под `asyncio.Lock` — медленный пир не сериализует всю комнату
- Защита от двойного `play` (атомарная проверка состояния)

## v0.2.2 — 2026-06-27

- Сравнение позиций синхронизации с допуском (`SYNC_TOLERANCE`) — нет вечного seek-цикла на дробной позиции
- Поздний участник пересинхронизируется (`add_user` сбрасывает готовность)
- Учёт длины ролика (`duration`) — концовка фильма больше не залипает
- Безопасный парсинг времени (NaN/Inf/строка/минус → 0) — кривой клиент не морозит комнату
- Изоляция сбоев рассылки (`return_exceptions`) — мёртвый сокет не роняет здоровых
- Валидация `video_url` (только `https://rezka.ag/`) в set_video / create / update / redirect
